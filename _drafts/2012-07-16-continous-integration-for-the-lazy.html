Getting started with Continous Integration is easy. In this post I am going to prove it, by showing the steps you need to take. Similar steps can be used for achieving Continous Integration on any platform - Java, Ruby, Python, etc. The kind of SCM repository used also makes little difference, because it is only a matter of installing an appropriate plugin. However, here I will be using:

  • Java
  • Maven
  • Jenkins/Hudson
  • Git (Github)

I do not want to explain the benefits of using Continous Integration, because there are already enough articles on this topic. But just to have a clear goal - we are trying to ensure that our application always compiles and successfully goes through the automatic tests.

First, you have to install and start the Jenkins/Hudson server. You have two options here:

  • download and install native package
  • download the jenkins.war and execute it with the command:


    java -jar jenkins.war

Next, open your browser and go to the location where Jenkins/Hudson is configured. With the default settings it will be:


http://localhost:8080

You should see a page similar to the following:

Before we add a new job, lets first add some essential plugins. Go to Manage Jenkins...

...and then click Manage plugins.

If you are working behind a proxy, it is important to configure it on the Advanced tab.

Finally, go to the Available tab, tick the following plugins:

  • Git Plugin
  • The Continuous Integration Game plugin
  • JavaTest Report Plugin
...and click the Install without restart
